Git 的基本使用方法

发表于 2022-01-09 更新于 2022-07-23 分类于 Linux笔记

1. Git 结构

  • 本地库：存放历史版本
  • 暂存区：用于临时存储
  • 工作区：写代码的地方

工作区 -> git add -> 暂存区 -> 本地库

2. Git 命令行操作

  1. 初始本地化库
    命令：git add
    效果：生成 .git 文件。其中存放本地库相关的子目录和文件

  2. 设置签名

    形式：用户名与Email地址
    作用：区分不同的开发人员的身份
    命令：

    1. 项目级别/仓库级别：仅在当前本地库范围内生效

      信息保存位置：./.git/config

    2. 系统用户级别

      信息保存位置：~/.gitconfig

  3. 基本操作

    • 状态查看
      git status：查看工作区，暂存区状态
    • 添加
      git add [file name] :将工作区的新建/修改添加到暂存区

    • 提交
      git commit -m “commmit message” [file name]
      将暂存区的内容提交到本地库

    • 查看历史记录
      git log :多屏显示控制方式(linux下：空格翻页，b向上翻页，q退出)
      git log —pretty=oneline
      git log —oneline
      git reflog: 长用

    • 前进后退
      基于索引值的操作[推荐]

      使用^符号：只能后退

      使用~符号：只能后退

      reset 命令三个参数对比
      —soft参数：仅仅在本地库移动HEAD指针
      —mixed参数：在本地库移动HEAD指针并且重置暂存区
      —hard参数：在本地库移动HEAD指针，重置暂存区，重置工作区

  4. 比较文件差异
    git diff [文件名]：将工作区中的文件和暂存区进行比较
    git diff [本地库中历史版本] [文件名]：将工作区中的文件和历史版本比较
