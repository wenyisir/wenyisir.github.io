接口技术

基本概念

在CPU和外设之间设置接口电路主要有以下一些原因:
（1）CPU与外设两者的信号线不兼容，在信号线功能定义、逻辑定义和时序关系上都不一致;
（2）两者的工作速度不兼容，CPU速度快，外设速度慢，相差好几个数量级。
（3）若不通过接口，由CPU直接对外设的操作进行控制，则CPU效率很低，会降低计算机性能。
（4）若外设直接由CPU控制，也会使外设的硬件结构依赖于CPU，对外设本身的发展不利。
采用了接口电路之后，不同厂商、不同时期的主机和外设就有可能连接使用了。

CPU与接口交换数据的控制方式

  1. 查询方式

    CPU传送数据前，检查外设是否准备好，若没有准备好，继续查其状态，直至外设准备好了，才进行数据传送。具体做法是在程序中安排一段由输入输出指令和测试指令及转移指令组成的程序段。CPU使用测试指令和条件转移指令循环检测设备完成准备工作的状态，一旦设备准备好，CPU执行传送指令，实现CPU与接口之间的数据传送。

  2. 中断方式

    在传送数据时，如果外设没有做好数据传送的准备，CPU可先执行与传送数据无关的其他指令。等到外设做好传送数据的准备后，再向CPU发出请求，CPU响应该请求，暂停正在运行的程序，转去执行用来进行数据传送的中断服务程序，完成数据传送后，再返回原来的程序继续执行。

  3. DMA方式

    数据的传送不经过CPU，由DMA控制器来实现内存与外设，或外设与外设的快速传送。

定时/技术技术和中断技术

定时/计数技术

定时/计数器接收信号源发送来的计数信号，到达约定的计数时，发出完成计数的信号。Intel系列的定时/计数器为8253和8254。

8254的内部逻辑结构

8254有三个完全相同的计数器，可以同时进行三组计数。8254是可编程接口，可以有多种工作方式，工作之前需要进行初始化。控制字寄存器用来保存初始化时的控制字；数据总线缓冲器用于将8254与系统数据总线$D_0$~$D_7$相连。读写控制逻辑接收CPU发来的读写信号和地址信号，来确定数据传输的方向是读出还是写入，并且选择所操作的寄存器。

概念

串行通信的概念

  • 并行通信：并行传输，以字节或者字为单位与I/O设备或者被控对象进行数据交换
  • 特点：传输速度快；硬件开销大；适合近距离传输
  • 串行通信：一位一位的进行数据传输
  • 特点：传输线少，成本低，适合远距离传送，缺点是速度慢

Git 的基本使用方法

发表于 2022-01-09 更新于 2022-07-23 分类于 Linux笔记

1. Git 结构

  • 本地库：存放历史版本
  • 暂存区：用于临时存储
  • 工作区：写代码的地方

工作区 -> git add -> 暂存区 -> 本地库

2. Git 命令行操作

  1. 初始本地化库
    命令：git add
    效果：生成 .git 文件。其中存放本地库相关的子目录和文件

  2. 设置签名

    形式：用户名与Email地址
    作用：区分不同的开发人员的身份
    命令：

    1. 项目级别/仓库级别：仅在当前本地库范围内生效

      信息保存位置：./.git/config

    2. 系统用户级别

      信息保存位置：~/.gitconfig

  3. 基本操作

    • 状态查看
      git status：查看工作区，暂存区状态
    • 添加
      git add [file name] :将工作区的新建/修改添加到暂存区

    • 提交
      git commit -m “commmit message” [file name]
      将暂存区的内容提交到本地库

    • 查看历史记录
      git log :多屏显示控制方式(linux下：空格翻页，b向上翻页，q退出)
      git log —pretty=oneline
      git log —oneline
      git reflog: 长用

    • 前进后退
      基于索引值的操作[推荐]

      使用^符号：只能后退

      使用~符号：只能后退

      reset 命令三个参数对比
      —soft参数：仅仅在本地库移动HEAD指针
      —mixed参数：在本地库移动HEAD指针并且重置暂存区
      —hard参数：在本地库移动HEAD指针，重置暂存区，重置工作区

  4. 比较文件差异
    git diff [文件名]：将工作区中的文件和暂存区进行比较
    git diff [本地库中历史版本] [文件名]：将工作区中的文件和历史版本比较
